Classes and Objects

What are classes?

A class is a certain data structure that allows the programmer to save code and keep the program simpler. It contains data, as well as certain functions, which is common across all instances of the class. By doing this, the programmer can quickly reuse code built earlier to extend the capabilities of their class, and know exactly what the class can or cannot do. This allows for more compact and modular code as classes simplify the production process by reusing the code already written. Classes are what make up object oriented programming, a it revolves around using classes as the main way to write code.

Code Example

Within this code example, we define a class for colour. This class contains information such as the pattern of the colours, and the primary and secondary colours. It also has a function to return a small description of the colour, which is used in the main code to describe the colour.

Constructors

One part of classes is the constructor, which is what is what is used whenever a class is created. It runs code that sets up the class correctly, assigning all variables, and in some cases generating information needed for the class to function properly.

Documentation

In C#, Microsoft recommends using XML style documentation, which you have been seeing throughout the code, as well as normal comments. Normal comments are defined as in many other languages using //. The programmer can then write the comment, helping others understand their code. For XML style documentation, there are tags which are used to describe various things. The most common type used is <summary> which is used to provide a small summary of what code does. Other types that are commonly used are <param>, and <returns> which describes parameters and the return values of functions. These can be used by other programmers to find out how to use the function properly.

Code Example

using System.Text.Json;

namespace Data_Structures.classes;

/// <summary>
///     Stores colour information (NON RGB)
/// <summary>
public class Colour
{
    /// <summary>
    ///     Class constructor
    /// <summary>
    /// <param name="pattern">Pattern of the colours (ex. spotted)</param>
    /// <param name="primary">Primary colour </param>
    /// <param name="secondary">Secondary colour</param>
    public Colour(string pattern, string primary, string secondary)
    {
        this.pattern = pattern;
        this.primary = primary;
        this.secondary = secondary;
    }

    /// <summary>
    ///     Stores the pattern of the colour as a string (ex. spotted)
    /// <summary>
    public string pattern { get; set; }

    /// <summary>
    ///     Stores the primary colour as a string
    /// <summary>
    public string primary { get; set; }

    /// <summary>
    ///     Stores the secondary colour as a string
    /// <summary>
    public string secondary { get; set; }

    /// <summary>
    ///     Gives a description of the colour in a string format
    /// <summary>
    /// lt;returns>A string containing the descriptionlt;/returns>
    public string Description()
    {
        if (primary != secondary)
            return $"{primary}-{secondary} {pattern}";
        return $"{primary}";
    }

    /// <summary>
    ///     Returns the cats information stored as a JSON.
    /// <summary>
    /// lt;returns>The cats information stored as a JSON, in a string.lt;/returns>
    public string ToJson()
    {
        return JsonSerializer.Serialize(this);
    }

    /// <summary>
    ///     Saves the colour information as a JSON
    /// <summary>
    public void SaveColour()
    {
        var fileName = @$".\data\colours\{primary}-{secondary}-{pattern}.json";
        var jsonString = ToJson();
        File.WriteAllText(fileName, jsonString);
    }
}

Inheritance

What is inheritance?

Inheritance is a concept which builds upon classes, as it uses classes as its base. In this case, a class will take variables and functions from another class, and extend it with additional information. This allows programmers to first design a much broader class, with basic functions that they know they will use a lot, and get more and more specific the further they extend the tree. Inhertiance also promotes the modularity of code, as it allows programmers to build specific extensions, splitting code across different sections. It also improves the maintainability of code, as programmers can change the base class (parent) and it will change across all the classes which inherit from it.

Code Example

Within this code example, we have a dog class, which extends from the base animal class. In this case, it adds on a variable called squirrelsChased, and a custom constructor which modified the base classes information. This allows the class to have custom information that differs from the base class of animal, and allows the reuse of code from before.

Documentation

Documentating this does not contain any special properties, and is pretty much the same as normal classes. This includes the XML documentation, as well as the normal comments within the code itself.

Code Example

using System.Text.Json;
using System.Text.Json.Serialization;

namespace Data_Structures.classes;

/// <summary>
///     A object that holds the characteristics of a dog, and some functions of a dog
/// <summary>
public class Dog : Animal
{
    /// <summary>
    ///     The class constructor
    /// <summary>
    public Dog()
    {
        guid = Guid.NewGuid();
        squirrelsChased = 0;
        skinType = "fur";
        species = "dog";
        // random number generator
        var rnd = new Random();

        string[] breeds =
        {
            "German Shepherd", "Bulldog", "Labrador Retriever", "Golden Retriever", "French Bulldog", "Shiba Inu",
            "Poodle", "Pitbull"
        };

        string[] sexs = { "Male", "Female" };

        var colours = new List<Colour>();
        // load colour information from file
        foreach (var file in Directory.GetFiles(@".\data\colours"))
        {
            // load json from file
            var jsonString = File.ReadAllText(@$".\{file}");
            // convert to class, and then save to colours list
            colours.Add(JsonSerializer.Deserialize<Colour>(jsonString)!);
        }

        // randomly generate stats for dog
        // get a random name from the user
        Console.WriteLine("What name would you like your dog?");
        name = Console.ReadLine()!;

        // choose random breed from list
        breed = breeds[rnd.Next(breeds.Length)];
        // choose random colour from list
        colour = colours[rnd.Next(colours.Count)];
        // choose random sex
        sex = sexs[rnd.Next(sexs.Length)];

        // choose random age
        age = rnd.Next(0, 10);
        // check if its a puppy
        if (age == 0)
        {
            // custom stats
            weight = rnd.NextDouble() * 1 + 2;
            height = rnd.NextDouble() * 5 + 3;
            length = rnd.NextDouble() * 6 + 4;
            speed = height * length * 0.05 - weight;
            // check if its a negative number, if it is set it to 1km/h
            if (speed <= 0) speed = 1;
        }
        else
        {
            // custom stats
            weight = rnd.NextDouble() * 3 + 3;
            height = rnd.NextDouble() * 10 + 23;
            length = rnd.NextDouble() * 10 + 50;
            speed = height * length * 0.05 - weight;
            // check if its a negative number, if it is set it to 1km/h
            if (speed <= 0) speed = 1;
        }
    }

    /// <summary>
    ///     The class constructor
    /// <summary>
    /// <param name="guid">Guid of the dog</param>
    /// <param name="name">Name of the dog</param>
    /// <param name="species">Species of the dog</param>
    /// <param name="colour">Colour of the dog</param>
    /// <param name="breed">Breed of the dog</param>
    /// <param name="sex">Sex of the dog</param>
    /// <param name="weight">Weight of the dog</param>
    /// <param name="height">Height of the dog</param>
    /// <param name="speed">Speed of the dog</param>
    /// <param name="length">Length of the dog</param>
    /// <param name="age">Age of the dog</param>
    /// <param name="skinType">Skin type of the dog</param>
    /// <param name="squirrelsChased">How many squirrels the dog has chased over its life</param>
    [JsonConstructor]
    public Dog(string name, string species, Colour colour, string breed, string sex, double weight, double height,
        double speed, double length, int age, string skinType, int squirrelsChased, Guid guid)
    {
        this.guid = guid;
        this.name = name;
        this.species = species;
        this.colour = colour;
        this.breed = breed;
        this.sex = sex;
        this.weight = weight;
        this.height = height;
        this.speed = speed;
        this.length = length;
        this.age = age;
        this.skinType = skinType;
        this.squirrelsChased = squirrelsChased;
    }


    /// <summary>
    ///     How mny squirrels the dog has chased over its lifetime
    /// <summary>
    public int squirrelsChased { get; set; }

    /// <summary>
    ///     Prints the animals sound (bark)
    /// <summary>
    public override void AnimalSound()
    {
        Console.WriteLine($@"{name} barks at you, and then gives you their toy.");
    }

    public override void SaveAnimal()
    {
        {
            var fileName = @$".\data\{species}\{guid.ToString()}.json";
            var jsonString = ToJson();
            File.WriteAllText(fileName, jsonString);
        }
    }
}

File Reading and Writing

JSON

One of the most common file formats to store information is to use a Javascript object notation (JSON) file. This file stores information in the same way that Javascript stores objects. In other languages, this would be most commonly known as a hashmap or a dictionary, where it stores information in a key, value pair. The information here can then be read in different languages, as many languages have a similiar data structure built in to store the information. This makes it one of the most versatile and useful file formats to store information, as it can be treated as a common language between the different programming languages as the information can be directly translated to the data structure within the language.

File Reading

In C#, file reading is done using the file library, built into C#, containing many methods to read information from a file. Most of the time, this information is returned as a string, which contains all the text requested. The data here can be then processed by a seperate library, or by code the programmer writes themselves.

File Writing

Similarly, the same library also deals with file writing. It does not process the information given to it into a specific format, but instead will just write it as a string into the file.

Code Example

Within the code example, we use the built in function to read and write files. For reading files, the information was stored within a JSON file, which contained the information within a class. First, the program reads this file, and then converts it into a string. This string is formatted in JSON, and so can be converted by another library into an object within C#, allowing us to manipulate and us it within our code. Similarly, we do the same thing to write a file, but in reverse. We first convert our class into a JSON object, which is then stored as a string. This string is then written raw into a file, with no processing occuring when it writes into the file.

Code Example

// load all default colours
var directory = @".\data\colours";
// loop through the files
foreach (var file in Directory.GetFiles(directory))
{
    // load file info as a string
    var jsonString = File.ReadAllText(@$".\{file}");
    // convert json to a class, and add to the list of colours
    colours.Add(JsonSerializer.Deserialize<Colour>(jsonString)!);
}
