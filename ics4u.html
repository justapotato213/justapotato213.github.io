UML Diagrams

What are they?

Unified Modeling Language (UML) is diagramming tool used to describe object oriented programs. UML class diagrams are a large part of the language, as it describes the links between different objects, as well the specifics of object. This includes its variables and functions, as well as whether they are public or private. The diagram allows programmers to get a visual view on how a program functions, and the specifics on what classes do. It includes syntax describing what classes use others, as well as what classes inherit from others. UML class diagrams can also help programmers plan out their projects, as it gives them a clear goal to keep moving towards.

Example Explanation

In this UML class diagram, it describes a program which uses various classes. Each of these classes are given a box, with a name at the top. Within each box, there are two sections, one with variables and one with functions. Each variable or function contains a - or + sign in front of it, indicating whether it is public or private. Each class also has a arrow, which represents the relationships between different classes. It indicates what classes use what, as well as which class is the child or parent of each. In this program, it laysout the basic usage of classes, and shows hwo each are linked and interconnected between them.
